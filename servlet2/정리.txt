1. 추상메소드 : 메소드의 설계도
- 부모 클래스는 메소드의 이름을 정해준다.
- 자식 클래스는 부모가 정해준 메소드를 구현한다.
- 같은 부모의 자식들을 사용방법(메소드)이 동일하다.
- 부모의 메소드는 추상 메소드 형식(구현하지 않는 메소드)
	 abstract class 부모 {
	 public abstract 너가만들 메소드();
	 }
	 
	 - 부모가 추상메소드 형식으로 자식에게 메소드 구현을 강제한다.
	 
	 
	 
	 
	 
	 2. 추상 클래스 
	 - 추상 메소드를 가지는 클래스 -> 객체 생성 불가능
	 	class 자식 extends 부모 {
	 	}
	 - 자식 클래스도 추상 클래스가 되어 객체 생성이 불가능 -> 에러 발생
	 - 부모가 추상 메소드로 메소드 이름을 정한다.
	 -> 상속받은 자식 클래스가 반드시 추상 메소드를 구현해야 한다.(안하면 에러)
	 
	추상클래스 요약 
 - 	추상 메소드를 가지는 클래스(표준에 해당하는 부모 클래스)
 - 객체를 생성할 수 없고 abstract 키워드를 사용하지 않을 경우 오류 발생 
	 
	 
	 3. 서블릿(Servlet)
	 - 웹서버에서 실행되는 자바 클래스 표준 - 표준은 이름이 정해져 있다. 
	 - get 방식 요청에 응답하는 doGet()
	 - post 방식 요청에 응답하는 doPost()
